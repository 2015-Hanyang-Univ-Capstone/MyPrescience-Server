    Recommendation Algorithms

    Item-Based Collaborative Filtering

  • 유사도 기반의 추천 알고리즘으로, User-Base 보다 Item-Base의 정확도가 더욱 높다.
  • 일반적인 CF는 추천 알고리즘의 쉬운 예로서, 같은 장르의 음악, 동일 아티스트의 음악 등의 추천이다.
  • Cold Start의 경우, 노래추천함에 있어 인기도있는 노래일수록 만족도가 높다.
  • 추천에 사용되는 데이터로, Ranking이 있는데, 이는 명시적/암시적으로 나눌 수 있다.
    명시적(explicit)은 사용자가 직접 점수를 입력하는 경우이고 암시적(implicit )점수는 점수는 입력하지 않으나, 조회하거나 리뷰없이 구매하는 경우를 뜻한다.
    명시적 점수는 사용자가 점수를 선정하는 경우가 드물기 때문에, 데이터의 편차가 심해져 추천에 합당하지 않은 경우가 많다.
    그에 비해 암시적 점수는 조회한 경우, 구매한 경우, 출력된 내용 보지도 않은 경우등을 따져 각각 Rating을 하여 명시적 점수보다는 많은 데이터의 점수를 얻을 수 있다.
  • 결론, 명시적 점수와 암시적 점수의 기준을 잘 세우고, 두 점수를 혼합하여 Ranking을 계산하는 것이 주 추천기능이고,!
    Cold Start에 경우, 사용자가 선택한 Genre에 따라 인기도 높은 순으로 노래를 추천한다. 혹은 유사한 Song, Artist 기반의 노래를 추천하는 방법으로 데이터를 일정 수 까지 추가한다.

    • Matrix Factorization Techniques

  • Item-Based Collaborative Filtering 는 메모리기반 추천서비스로, 데이터의 양이 많으면 많아질수록 메모리에 과부하가 걸리게 된다.
  • 이러한 데이터의 크기 (data dimensionality)를 해결하기 위한 기법으로 행렬 분해를 통해 P*Q = R 의 P, Q의 행렬로 나누어 행렬 메모리 값을 저장할 수 있다.
    노래의 갯수 : n , 유저의 수 : m 일때, n * m 의 데이터 크기에서 ( n + m ) * k의 수로 (k는 임의의 수) 크기를 줄일 수 있게 된다.
  • Latent Factor Models are based on matrix factorization.
  • The model is closely related to singular value decomposition (SVD)
  • probabilistic matrix factorization
